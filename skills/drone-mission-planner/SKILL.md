# 无人机任务规划 Skill

## 目标

把自然语言无人机作业需求转成可人工审批的任务计划。该 Skill 只生成 review-only 任务方案，不直接下发飞控命令。

## 输入示例

```text
明天上午 9 点让两架无人机巡检 A 区域的输电线路，重点检查杆塔和疑似异物。
```

## 执行流程

1. 解析任务目标、区域、无人机数量、时间窗口和巡检目标。
2. 查询示例地图上下文、起降点、障碍物和作业走廊。
3. 检查禁飞区和临时管制区冲突。
4. 获取天气和风速等飞行约束。
5. 生成多无人机航线分配、航点、载荷和动作。
6. 评估天气、禁飞区、电量、夜航等风险。
7. 输出人工审批用任务计划。

## 安全边界

- 不直接调用真实无人机飞控系统。
- 不输出可绕过审批的自动下发命令。
- 高风险任务必须人工确认禁飞区、天气、电量和现场安全条件。
- 后续接真实飞控时，应将下发动作放在单独 tool 中，并强制人工审批和审计。

## 面试讲法

这个 Skill 展示的是“业务 Agent 化”的方法：LLM 或自然语言入口只负责表达需求，真正的地图、天气、禁飞区、航线和风险判断都拆成确定性工具。Workflow 负责把工具编排成可审计链路，Trace 记录每一次工具调用，最终输出可审批计划而不是直接执行危险动作。
